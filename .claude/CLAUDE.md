# SyncMind Starter 项目规则

## 全局输出规范

### 🔥 强制规则：所有 Skill 输出必须经过人性化处理

所有 skills（除 `humanizer-zh` 本身外）在生成最终输出时，**必须**应用 `humanizer-zh` skill 的规则来处理文本，确保输出：

1. **去除 AI 味道** - 避免夸大、模糊归因、过度使用连接词等 AI 写作痕迹
2. **语言自然** - 像真实的人在说话，而不是机器生成
3. **直接精炼** - 直接陈述事实，不绕圈子

### 处理流程

```
Skill 生成内容 → humanizer-zh 规则处理 → 最终输出
```

### 适用的 Skills

以下 skills 的输出必须经过人性化处理：

- `/profile` - 客户档案
- `/sales-guide` - 销售指南
- `/requirements` - 需求文档
- `/plan-writer` - 解决方案
- `/spec-writer` - 技术规格
- `/insights` - 洞察分析

### humanizer-zh 核心规则速查

在生成任何文本时，牢记这 5 条核心原则：

1. **删除填充短语** - 去除开场白和强调性拐杖词
2. **打破公式结构** - 避免二元对比、戏剧性分段、修辞性设置
3. **变化节奏** - 混合句子长度。两项优于三项。段落结尾要多样化
4. **信任读者** - 直接陈述事实，跳过软化、辩解和手把手引导
5. **删除金句** - 如果听起来像可引用的语句，重写它

### 需要避免的 AI 词汇

❌ 此外、与……保持一致、至关重要、深入探讨、强调、持久的、增强、培养、获得、突出（动词）、相互作用、复杂/复杂性、关键（形容词）、格局（抽象名词）、关键性的、展示、织锦（抽象名词）、证明、强调（动词）、宝贵的、充满活力的

### 需要避免的模式

❌ "这不仅仅是……而是……" - 否定式排比
❌ "X、Y 和 Z" - 三段式法则过度使用
❌ "作为/充当/标志着" - 系动词回避
❌ 破折号过度使用
❌ 过度强调意义、遗产和更广泛的趋势

## 项目结构

```
syncmind-starter/
├── .claude/
│   ├── CLAUDE.md          # 全局配置（本文件）
│   └── skills/            # Skills 目录
│       ├── humanizer-zh/  # 人性化处理（核心依赖）
│       ├── profile/       # 客户档案
│       ├── sales-guide/   # 销售指南
│       ├── requirements/  # 需求文档
│       ├── plan-writer/   # 解决方案
│       ├── spec-writer/   # 技术规格
│       └── insights/      # 洞察分析
├── docs/
│   ├── customer/          # 当前客户数据
│   ├── customers/         # 客户列表
│   └── plan/              # 方案文档
└── src/                   # 应用代码
```

## 其他规范

### 语言要求

- 代码注释：英文
- 输出文档：中文
- 技术术语：保留英文原文，必要时加中文注释

### 文档格式

- 使用 Markdown 格式
- 架构图使用 Mermaid 语法
- JSON 数据结构需要有 TypeScript 类型定义
